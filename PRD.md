# PRD：命令列 To-Do List 應用程式（Python）

## 1. 產品概述
本產品是一個以 Python 開發的命令列（CLI）待辦清單工具，讓使用者可以快速管理每日任務。

核心目標：
- 以最小學習成本完成任務管理。
- 資料可持久化保存於本地 JSON 檔案。
- 提供清楚、可預測的命令與輸出格式。

## 2. 目標使用者
- 初學者或個人開發者。
- 偏好在終端機工作流程中管理待辦事項的使用者。

## 3. 功能範圍（MVP）
### 3.1 新增任務
- 使用命令新增一筆待辦任務。
- 任務至少包含：`id`、`title`、`completed`。

### 3.2 列出任務
- 顯示所有任務。
- 每筆任務需展示：ID、狀態（完成/未完成）、標題。

### 3.3 標記完成
- 透過任務 ID 將指定任務標記為完成。
- 若 ID 不存在，提供明確錯誤訊息。

### 3.4 刪除任務
- 透過任務 ID 刪除指定任務。
- 若 ID 不存在，提供明確錯誤訊息。

## 4. 非功能需求
- 使用 Python 3.10+。
- JSON 檔案作為唯一資料來源，不需資料庫。
- 檔案讀寫需有錯誤處理（檔案不存在、JSON 格式異常）。
- 程式碼須具備型別註記與基本單元測試。

## 5. 命令設計
建議命令介面如下：

- `python -m todo_cli add "買牛奶"`
- `python -m todo_cli list`
- `python -m todo_cli complete 1`
- `python -m todo_cli delete 1`

可選參數：
- `--file <path>`：指定 JSON 儲存路徑（預設為 `tasks.json`）。

## 6. 資料模型
JSON 結構：

```json
{
  "tasks": [
    {
      "id": 1,
      "title": "買牛奶",
      "completed": false
    }
  ],
  "next_id": 2
}
```

欄位說明：
- `tasks`：任務陣列。
- `next_id`：下一筆任務應分配的 ID。

## 7. 例外與邊界情境
- 新增空字串標題：拒絕並提示使用者。
- 標記完成/刪除不存在 ID：回傳非 0 狀態碼並顯示錯誤。
- JSON 檔案不存在：自動初始化預設資料。
- JSON 檔損毀：中止操作並顯示可理解的錯誤訊息。

## 8. 驗收標準
- 可成功新增任務並持久化到 JSON。
- `list` 可正確顯示任務與狀態。
- `complete <id>` 可更新指定任務狀態。
- `delete <id>` 可移除指定任務。
- 以上操作在重啟程式後仍保留結果。
- 有自動化測試覆蓋主要流程與常見錯誤情境。
